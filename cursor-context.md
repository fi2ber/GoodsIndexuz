PROJECT OVERVIEW — READ CAREFULLY AND FOLLOW STRICTLY

You are working on a B2B wholesale export website for agricultural products from Uzbekistan.
This is NOT an e-commerce website.
This is NOT a consumer-facing product.

The website represents a real export supplier working with:
- importers
- distributors
- wholesalers
- contract buyers

The primary goal is to build TRUST and enable PRICE REQUESTS via Telegram or inquiry form.

────────────────────────────────────────
1. BUSINESS MODEL
────────────────────────────────────────

- B2B only
- Wholesale / export / contract supply
- No online payments
- No cart
- No checkout
- No visible prices
- Communication-driven sales

User journey:
1. Visitor understands company credibility
2. Visitor explores product range
3. Visitor opens a product
4. Visitor requests price via Telegram or form
5. Inquiry is logged in admin panel

────────────────────────────────────────
2. TARGET AUDIENCE
────────────────────────────────────────

- Professional buyers
- Procurement managers
- Import/export companies
- Distributors inside Uzbekistan
- People who value clarity, documents, specs, reliability

UX must be:
- simple
- calm
- professional
- intuitive even for non-technical users

────────────────────────────────────────
3. DESIGN & UX PHILOSOPHY
────────────────────────────────────────

Design style:
- Premium minimalism
- Inspired by Stripe, Linear, Framer, Webflow B2B awards
- Large typography
- Generous spacing
- Strong visual hierarchy
- No visual noise

Strictly avoid:
- Popups
- Banners
- Countdown timers
- Sales hype
- Dark patterns
- Consumer e-commerce UI

Mobile-first is mandatory.
Telegram-first UX is mandatory.

────────────────────────────────────────
4. CONTENT & TONE
────────────────────────────────────────

Tone of voice:
- Calm
- Confident
- Factual
- Export-grade
- Professional

Content rules:
- No marketing exaggerations
- No emotional language
- No promises without proof
- No invented certifications
- No “best”, “number one”, “cheap”, etc.

All texts must feel suitable for:
- commercial proposals
- contracts
- international trade discussions

────────────────────────────────────────
5. LANGUAGES
────────────────────────────────────────

Supported languages:
- English (primary)
- Russian (secondary)

All user-facing content must exist in both languages.
Translations must be professional, not literal.

────────────────────────────────────────
6. PRODUCT LOGIC
────────────────────────────────────────

Each product represents an export position.

Every product MUST include:
- Product name
- Category
- Variety / type
- Origin (region in Uzbekistan)
- Packaging options
- MOQ
- Shelf life
- Export readiness statement

Product pages must focus on:
- specifications
- consistency
- logistics readiness

No prices. Ever.

────────────────────────────────────────
7. AI ASSISTANT ROLE
────────────────────────────────────────

AI is used ONLY as an assistant.

AI can:
- generate RU / EN product descriptions
- generate specification summaries
- generate Telegram-ready inquiry text
- help admin fill structured fields faster

AI must:
- follow B2B export tone
- avoid marketing clichés
- never invent facts, certifications or guarantees

AI must NOT:
- replace structured data
- generate unsupported claims

────────────────────────────────────────
8. ADMIN PANEL PHILOSOPHY
────────────────────────────────────────

Admin panel goals:
- Fast product creation
- Minimal screens
- Clear workflows
- AI as a helper, not core logic

Admin features:
- Product CRUD
- AI generate button
- Inquiry list
- No complex CMS patterns

────────────────────────────────────────
9. TECH STACK (MANDATORY)
────────────────────────────────────────

Frontend:
- Next.js 14 (App Router)
- TypeScript
- Tailwind CSS
- shadcn/ui
- Framer Motion (minimal, subtle)

Backend:
- PostgreSQL (database, auth)

Architecture rules:
- Server Components by default
- Strong typing everywhere
- Reusable components
- Config-driven structure
- No hardcoded content

────────────────────────────────────────
10. PERFORMANCE, SEO & QUALITY
────────────────────────────────────────

- Fast loading
- Semantic HTML
- SEO-friendly URLs
- Export-ready metadata
- Optimized images
- No console errors
- No build warnings

────────────────────────────────────────
11. DECISION PRIORITY
────────────────────────────────────────

If there is any uncertainty, always prioritize:

1. Trust
2. Clarity
3. Simplicity
4. Performance
5. Aesthetics

Never sacrifice trust for visual effects.
